活动（Activity_2）

2019-04-03Android

一、活动的生命周期

​ Android使用任务(task)来管理活动的，一个任务就是一组存放在栈里的活动的结合，这个栈被称作返回栈(Back Stack)。

  • 活动状态

    1. 运行状态

      位于返回栈的栈顶。

    2. 暂停状态

      不再处于返回栈的栈顶，但仍然可见。

    3. 停止状态

      不再处于返回栈的栈顶，并且完全不可见。

    4. 销毁状态

      从返回栈中移除后，就变成了销毁状态。

  • 活动生存期

    • onCreate() 。这个方法在活动第一次创建的时候调用。
    • onStart()。这个方法在活动由不可见变为可见的时候调用。
    • onResume()。这个方法在活动准备好和用户进行交互的时候调用。
    • onPause()。这个方法在系统准备去启动或者恢复另一个活动的时候调用。
    • onStop()。这个方法在活动完全不可见的时候调用。
    • onDestroy()。这个方法在活动被销毁之前调用。
    • onRestart()。在这个方法在活动由停止状态变为运行状态之前调用。

    1. 完整生存期

      活动在onCreate()方法和onDestroy()方法之间所经历的，就是完整生存期。

    2. 可见生存期

      活动在onStart()方法和onStop()方法之间所经历的，就是可见生存期。

    3. 前台生存期

      活动在onResume()方法和onPause()方法之间所经历的，就是前台生存期。

总结：在一个活动中，如果开启了其他活动，并且其他活动使原活动不可见，在这个过程中依次调用了onPause()方法和onStop()方法，在返回原活动的过程中依次调用了onRestart()方法、onStart()方法和onResume()方法；如果开启的活动为对话框类型的，则会调用onPause()方法，返回时调用onResume()方法。在创建一个活动的时候会首先调用onCreate()方法、onStart()方法和onResume()方法。在退出活动的时候调用onDestroy()方法。

二、活动回收

在活动中添加如下代码：

// 如果活动被回收，通过该方法保存活动中的临时数据
// 存入数据
@Override
protected void onSaveInstanceState(Bundle outState) {
    super.onSaveInstanceState(outState);
    String data = "data";
    outState.putString("data_key", data);
}

// 在onCreate()方法中添加下述代码
// 取出数据
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    if(savedInstanceState != null) {
        String data = savedInstanceState.getString("data_key");
    }
}

补充：相应地，可以将Bundle和Intent结合起来传送数据，即先将数据保存到Bundle对象中，再将Bundle对象保存到Intent对象中。

三、活动的启动模式

  • standard

    每次都创建活动的新实例。

  • singleTop

    如果活动在返回栈的栈顶，则不创建新实例，否则创建新实例。

  • singleTask

    如果活动在返回栈中，则不创建新实例，直接返回到栈顶。

  • singleInstance

    启用一个新的返回栈来管理活动。

    注：在 Androidanifest.xml 中注册活动时，在 标签中加入android:launchMode。

四、活动使用小技巧

  1. 知晓当前是哪一个活动

    ​ 新建一个Java类，继承AppCompatActivity，然后让活动继承此类，在类中重写onCreate()方法，并在方法中添加Log.d(“Activity”, getClass().getSimpleName())。

  2. 随时随地退出程序

    ​ 新建一个ActivityCollector类作为活动管理器。

    public class ActivityCollector {
    public static List<Activity> activities = new ArrayList<>();

    public static void addActivity(Activity activity) {
        activities.add(activity);
    }

    public static void removeActivity(Activity activity) {
        activities.remove(activity);
    }

    public static void finishAll() {
        for(Activity activity : activities) {
            if(!activity.isFinishing()) {
                activity.finish();
            }
        }
    }
}

  1. 启动活动最佳写法

    在活动中添加actionStart(Context context, String data)。其中数据可根据需求添加。Activity为被启动活动的类名，该方法写在被启动活动中。

    public static void actionStart(Context context, String data) {
    Intent intent = new Intent(context, Activity.class);
    intent.putExtra("param", data);
    context.startActivity(intent);
}
