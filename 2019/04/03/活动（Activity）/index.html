活动（Activity）

2019-04-03Android

一、活动(Activity)

1. 手动创建活动(.java文件)

  1. 在Android Studio中app/src/main/java/com.example.activitytest的目录下创建：
    Activity->Empty Activity。不勾选Generate Layout File 和 Launcher Activity 这两个选项，分别表示为活动创建一个对应的布局文件，和设置为当前项目的主活动。具体代码内容如下：

    public class MainActivity extends AppCompatActivity {
 @Override
 protected void onCreate(Bundle savedInstanceState) {
     super.onCreate(savedInstanceState);      // 表示创建
     setContentView(R.layout.main_layout);    // 加载布局
 }
}

    setContentView()传入的参数为布局文件的id。

    1. 创建布局(layout.xml文件)

      <?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
android:orientation="vertical"
android:layout_width="match_parent"
android:layout_height="match_parent">
<Button
  android:id="@+id/button_1"
  android:layout_width="wrap_content"
  android:layout_height="match_parent"
  android:text="Button" />
</LinearLayout>

      引用id：@id/name
      定义id：@+id/name

    2. 注册活动(AndroidManifest.xml)

    <?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
 package="com.example.zhang.myapplication">
 <application
     android:allowBackup="true"
     android:icon="@mipmap/ic_launcher"
     android:label="@string/app_name"
     android:roundIcon="@mipmap/ic_launcher_round"
     android:supportsRtl="true"
     android:theme="@style/AppTheme">
     <!--注册活动-->
     <!--android:label表示标题栏中的内容-->
     <activity android:name=".MainActivity"
         android:label="Main Activity">
         <!--配置主活动的方法-->
         <intent-filter>
             <action android:name="android.intent.action.MAIN" />
             <category android:name="android.intent.category.LAUNCHER" />
         </intent-filter>
     </activity>
 </application>
</manifest>

    活动名写 .Activity 的原因是packae属性指定了程序的包名。

二、活动中使用

1. Toast

Toast.makeText(Activity.this, “msg”, Toast.LENGTH_SHORT).show();

           Button button = (Button)findViewById(R.id.button_1);
           button.setOnClickListener(new View.OnClickListener() {
               @Override
               public void onClick(View v) {
                   Toast.makeText(MainActivity.this,
                                  "You clicked button",
                                  Toast.LENGTH_SHORT).show();
               }
           });

在这段代码中，通过findViewById()方法获得在布局文件中定义的元素，传入参数为R.id.button_1。 然后通过setOnClickListener()方法来写按钮的监听事件。

2.Menu

  1. 首先需要在res目录下新建menu文件夹，然后新建Menu resource file，最后在xml文件中编写布局。

    <?xml version="1.0" encoding="utf-8"?>
<menu xmlns:android="http://schemas.android.com/apk/res/android">
 <item
     android:id="@+id/add"
     android:title="Add" />
 <item
     android:id="@+id/remove"
     android:title="Remove" />
</menu>

  2. 在活动中需要重写onCreateOptionsMenu()方法和onOptionsItemSelected()方法。快捷键（ctrl+O）。

  • oncreateOptionsMenu()

      // 用于显示菜单
  @Override
public boolean onCreateOptionsMenu(Menu menu) {
    getMenuInflater().inflate(R.menu.main,menu);
    return true;
}

    getMenuInflater()方法是得到MenuInflater对象，再调用对象的inflate()方法给当前活动创建菜单。inflate() 中两个参数，第一个表示资源文件，第二个表示菜单项添加到具体的 Menu 对象中，返回值true表示显示，false表示不显示。

  • onOptionsItemSelected

      @Override
  public boolean onOptionsItemSelected(MenuItem item) {
      switch(item.getItemId()) {
          case R.id.add:
              Toast.makeText(MainActivity.this,"Add",
                             Toast.LENGTH_LONG).show();
              break;
          case R.id.remove:
              Toast.makeText(MainActivity.this,"Remove",
                             Toast.LENGTH_LONG).show();
              break;
          default:
      }
      return true;
  }

3.Intent

  • 显式Intent

    Button button_2 = (Button)findViewById(R.id.button_2);
button_2.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        Intent intent = new Intent(MainActivity.this, FirstActivity.class);
        startActivity(intent);
    }
});

    显式调用， Intent(Context packageContext, Class<?> cls) ，第一个参数表示启动活动的上下文，第二个参数表示要启动的目标活动。

  • 隐式Intent

    在AndroidManifest文件中

     <activity
    android:name=".SecondActivity"
    android:label="SecondActivity">
     <intent-filter>
         <action android:name="com.example.zhang.myapplication.ACTION_START" />
         <category android:name="android.intent.category.DEFAULT" />
         <category android:name="com.example.zhang.myapplication.myCategory" />
     </intent-filter>
</activity>

    如果为<category android:name="android.intent.category.DEFAULT" /> 在活动中就不需要使用 addCategory() 方法， startActivity() 方法会自动将这个category添加到 Intent 中。

    在FirstActivity中

    Button button_1 = (Button)findViewById(R.id.button_first_1);
button_1.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        Intent intent = new Intent("com.example.zhang.myapplication.ACTION_START");
        intent.addCategory("com.example.zhang.myapplication.myCategory");
        startActivity(intent);
    }
});

    更多的隐式Intent用法：

    // Intent.ACTION_VIEW 表示Android系统的内置动作
Intent intent = new Intent(Intent.ACTION_VIEW);
// setData()方法接受一个Uri对象，用于指定当前Intent正在操作的数据
intent.setData(Uri.parse("http://www.baidu.com"));
startActivity(intent);

    <!--
    在AndroidManifest.xml文件的 <intent-filter>标签中配置<data>标签，用于更精确地指定当前活动能够响应什么类型的数据。
-->
<activity
    android:name=".ThirdActivity">
    <intent-filter>
        <action android:name="android.intent.action.VIEW" />
        <category android:name="android.intent.category.DEFAULT" />
        <data android:scheme="http" />
    </intent-filter>
</activity>

    标签还可以配置以下内容：

    • android:scheme 用于指定数据的协议部分。
    • android:host 用于指定数据的主机名部分。
    • android:port 用于指定数据的端口部分，一般紧随在主机名之后。
    • android:path 用于指定主机名和端口之后的部分。
    • android:mimeType 用于指定可以处理的数据类型，允许使用通配符的方式进行指定。
    // Intent.ACTION_DIAL，内置动作（拨号）
// tel，协议，表示拨打电话
Intent intent = new Intent(Intent.ACTION_DIAL);
intent.setData(Uri.parse("tel:10086"));
startActivity(intent);
// 补充geo表示地理位置

  • Intent 传递数据

    1. 向下一个活动传递数据

      Intent intent = new Intent(MainActivity.this, FirstActivity.class);
intent.putExtra("Main data", "Hello FirstActivity");
startActivity(intent);

      Intent 通过方法 putExtra() 为下一个活动传递数据。第一个参数是键，第二个参数是传递的数据。

      在下一个活动中利用 getIntent() 方法获得启动 Activity 的 Intent ，然后通过 getStringExtra() 取出数据。

      Intent intent = getIntent();
String msg = intent.getStringExtra("Main data");
// 补充：String类型就用getStringExtra()，int类型用getIntExtra()，以此类推。

    2. 返回数据给上一个活动

      // 在第一个活动中，使用 startActivityForResult() 方法开启下一个活动
// 第一个参数为 Intent 对象，第二个表示请求码
// startActivityForResult(Intent intent, int requestCode);
Intent intent = new Intent(SecondActivity.this,ThirdActivity.class);
startActivityForResult(intent,1);
// 同时需要重写 onActivityResult() 方法
protected void onActivityResult(int requestCode, int resultCode, Intent intent){
    switch (requestCode) {
        case 1:
            if(resultCode == RESULT_OK) {
                // 通过 getStringExtra() 方法得到数据
                String data = intent.getStringExtra("return");
                Toast.makeText(SecondActivity.this, data,
                               Toast.LENGTH_LONG).show();
            }
            break;
    }
}

      // 在第二个活动中，需要返回数据给第一个活动，通过 putExtra() 方法存放数据
// 然后通过 setResult() 方法向上一个活动返回数据
// 第一个参数为返回处理结果，一般为RESULT_OK和RESULT_CANCELED
// 第二个参数为带有数据的 Intent 的对象
Intent intent = new Intent();
intent.putExtra("return", "Hello Second Activity");
setResult(RESULT_OK, intent);
finish();

// 以上代码是写在按钮的响应时间中，如果用户使用返回键退出活动，需要重写 onBackPressd() 方法
// 数据传递的方法相同
@Override
public void onBackPressed() {
    Intent intent = new Intent();
    intent.putExtra("back","Hi, Second Activity");
    setResult(RESULT_CANCELED, intent);
    finish();
}
