Android Studio项目文件说明

2019-04-03Android

一、Android Studio的项目文件

  1. gradle和.idea

    Android Studio 自动生成的一些文件。

  2. app
    项目中的代码、资源等内容几乎都是放置在该目录下。

    1. build

      编译时自动生成的文件。

    2. libs

      在项目中用到了第三方jar包，就放在该目录下。

    3. androidTest

      用来编写Android Test测试用例的。

    4. java

      用来编写java代码的目录。

    5. res

      drawable用来存放图片资源，layout用来存放布局资源，values用来存放字符串资源，mipmap存放图标资源。

    6. AndroidManifest.xml

      整个Android项目的配置文件，四大组件需要在这个文件中注册。

    7. test

      此处用来编写Unit Test测试用例的。

    8. .gitgnore

      这个文件用于将app模块内的指定的目录或文件排除在版本控制之外，和外层的.gitgnore文件类似。

    9. app.iml

      IntelliJ IDEA项目自动的生成的文件。

    10. build.gradle

      app模块的gradle构建脚本，这个文件会指定很多项目构建相关的配置。

    11. proguard-rules.pro

      这个文件用于指定项目代码的混淆规则。

  3. build

    在编译时自动生成的文件。

  4. gradle

    存放gradle wrapper的配置文件。

  5. .gitgnore

    这个文件是用来将指定的目录或文件排除在版本控制之外的。

  6. build.gradle

    这是项目的全局构建脚本，通常不需要修改。

  7. gradle.properties

    这个文件是全局的gradle配置文件，影响项目中所有的gradle编译脚本。

  8. gradlew和gradlew.bat

    这两个文件用来在命令行界面中执行gradle命令的。

  9. HelloWorld.iml

    IntelliJ IDEA 项目自动生成的，用于标识这是一个IDEA项目。

  10. local.properties

    这个文件用于指定本机中Android SDK路径，自动生成的。

  11. settings.gradle

    用于指定项目中所有引入的模块。

二、重要文件说明

1、AndroidManifest.xml

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:roundIcon="@mipmap/ic_launcher_round"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>

intent-filter中的两行代码很重要，表示MainActivity是这个项目的主活动。

2、项目中的build.gradle

通常不需要修改。

buildscript {

    repositories {
        google()
        jcenter()  // 代码托管仓库
    }
    dependencies {
        classpath 'com.android.tools.build:gradle:3.2.1'  // 声明了Gradle插件
    }
}
allprojects {
    repositories {
        google()
        jcenter()
    }
}

task clean(type: Delete) {
    delete rootProject.buildDir
}

3、app中的gradle.build

apply plugin: 'com.android.application'  // 表示这是一个应用程序模块
// com.android.library 表示这是一个库模块
android {
    compileSdkVersion 28  // 指定项目的编译版本
    defaultConfig {
        applicationId "com.example.zhang.application"  // 指定项目的包名
        minSdkVersion 15     // 项目最低兼容的Android系统版本
        targetSdkVersion 28  // 目标Android系统版本
        versionCode 1        // 项目的版本号
        versionName "1.0"    // 项目的版本名
        testInstrumentationRunner "android.support.test.runner.AndroidJUnitRunner"
    }
    buildTypes {
        release {
            minifyEnabled false  // 是否对项目的代码进行混淆
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguardrules.pro'   // 混淆使用的规则文件
        }
    }
}

dependencies {
    // implementation用于引入一些库，之前为compile
    implementation fileTree(dir: 'libs', include: ['*.jar'])
    implementation 'com.android.support:appcompat-v7:28.0.0'
    implementation 'com.android.support.constraint:constraint-layout:1.1.3'
    testImplementation 'junit:junit:4.12'
    androidTestImplementation 'com.android.support.test:runner:1.0.2'
    androidTestImplementation 'com.android.support.test.espresso:espresso-core:3.0.2'
}

三、Android的Log工具

  • Log.v()，对应级别为verbose。
  • Log.d()，对应级别为debug。
  • Log.i()，对应级别为info。
  • Log.w()，对应级别为warn。
  • Log.e()，对应级别为error。

    Log.d()传入两个参数：tag，msg；tag一般为当前的类名，msg为想打印的内容。
