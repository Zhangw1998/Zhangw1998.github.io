Activity

1. 生命周期

Android使用任务(task)来管理活动的，一个任务就是一组存放在栈里的活动的结合，这个栈被称作返回栈(Back Stack)。

1.1 正常情况的生命周期

    1. onCreate : Activity正在被创建

该方法在活动第一次创建的时候调用，一般用于加载布局、初始化Activity所需的数据等等。

    1. onStart : Activity正在被启动，已经显示出来，但没有出现在前台

该方法在活动由不可见变为可见的时候调用，一般情况下在执行完onCreate就会被调用

    1. onResume : Activity已可见，出现在前台，可与用户进行交互

该方法在活动准备好和用户进行交互的时候调用，一般在执行完onStart后就会被调用

    1. onPause ： Activity正在停止

该方法在系统准备去启动或者恢复另一个活动的时候调用，可以执行一些不太耗时的操作

    1. onStop : Activity即将停止

方法在活动完全不可见的时候调用，可执行一些稍微重量级的回收工作（不能太耗时）

    1. onDestroy : Activity即将被销毁

该方法在活动被销毁之前调用，可执行一些回收工作和资源释放

    1. onRestart : Activity正在重新启动

该方法在活动由停止状态变为运行状态之前调用

1.2 异常情况的生命周期

  • 系统配置发生改变

例如，设备由竖屏变为了横屏、修改了系统的语言等等，这种情况下会重新创建活动。
有个比较重要的方法onSaveInstanceState和onRestoreInstanceState。onSaveInstanceState用于保存一些比较重要的临时数据，当活动被重新创建时可以从onRestoreInstanceState中取出(推荐)，也可以通过onCreate方法中的参数获取。

// 该方法在onDestroy之前调用
@Override
protected void onSaveInstanceState(Bundle outState) {
    super.onSaveInstanceState(outState);
    // 在此处通过outState.putXXX方法存储临时数据

}

// 该方法在onResume之前调用
@Override
protected void onRestoreInstanceState(Bundle savedInstanceState) {
    super.onRestoreInstanceState(savedInstanceState);
    // 在此处通过调用savedInstanceState.getXXX方法获取临时数据

}

通过在AndroidManifest.xml中为标签设置属性configChanges，可让属性改变时，Activity不会被重新创建。但是会调用onConfigurationhanged方法（不推荐）
常用属性：orientation、screenSize

// 在此处处理系统配置改变后的情况
@Override
public void onConfigurationChanged(Configuration newConfig) {
    super.newConfig);
}
  • 资源内存不足导致低优先级的Activity被杀死

优先级(高到低)：

  1. 前台Activity
  2. 可见非前台Activity
  3. 后台Activity
    生命周期同系统配置发生改变
  • 多窗口模式

Android 7.0中加入了多窗口模式，当进入多窗口模式时，会调用onDestroy销毁活动然后重新创建，因此如果有比较重要的临时数据需要保存到Bundle中。

<!--禁用多窗口模式-->
android:resizeableActivity="false"

1.3 活动状态

  1. 运行状态

    位于返回栈的栈顶。

    1. 暂停状态

      不再处于返回栈的栈顶，但仍然可见。

    2. 停止状态

      不再处于返回栈的栈顶，并且完全不可见。

    3. 销毁状态

      从返回栈中移除后，就变成了销毁状态。

1.4 活动生存期

  1. 完整生存期

    活动在onCreate()方法和onDestroy()方法之间所经历的，就是完整生存期。

  2. 可见生存期

    活动在onStart()方法和onStop()方法之间所经历的，就是可见生存期。

  3. 前台生存期

    活动在onResume()方法和onPause()方法之间所经历的，就是前台生存期。

1.5 生命周期代码

public class MainActivity extends AppCompatActivity {

    private static final String TAG = "MainActivity";

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        Log.d(TAG, "onCreate: ");
    }

    @Override
    public void onStart() {
        super.onStart();
        Log.d(TAG, "onStart: ");
    }

    @Override
    public void onResume() {
        super.onResume();
        Log.d(TAG, "onResume: ");
    }

    @Override
    public void onPause() {
        super.onPause();
        Log.d(TAG, "onPause: ");
    }

    @Override
    public void onStop() {
        super.onStop();
        Log.d(TAG, "onStop: ");
    }

    @Override
    public void onDestroy() {
        super.onDestroy();
        Log.d(TAG, "onDestroy: ");
    }

    @Override
    public void onRestart() {
        super.onRestart();
        Log.d(TAG, "onRestart: ");
    }

    @Override
    protected void onSaveInstanceState(Bundle outState) {
        super.onSaveInstanceState(outState);
        Log.d(TAG, "onSaveInstanceState: ");
    }

    @Override
    protected void onRestoreInstanceState(Bundle savedInstanceState) {
        super.onRestoreInstanceState(savedInstanceState);
        Log.d(TAG, "onRestoreInstanceState: " + data);
    }

    @Override
    public void onConfigurationChanged(Configuration newConfig) {
        super.onConfigurationChanged(newConfig);
        Log.d(TAG, "onConfigurationChanged: ");
    }

    @Override
    protected void onNewIntent(Intent intent) {
        super.onNewIntent(intent);
        Log.d(TAG, "onNewIntent: ");
    }

}
  • 总结

  1. 打开App -> onCreate -> onStart -> onResume
    进入下一个活动(使该活动完全不可见) -> onPause -> onStop
    返回本活动 -> onRestart -> onStart -> onResume

  2. 打开App -> onCreate -> onStart -> onResume
    进入下一个活动(该活动可见) -> onPause
    返回本活动 -> onResume

  3. 打开App -> onCreate -> onStart -> onResume
    进入多窗口模式 -> onPause -> onStop -> onDestroy -> onCreate -> onStart -> onResume
    退出多窗口模式 -> onPause -> onStop -> onDestroy -> onCreate -> onStart -> onResume

2. 启动模式

2.1 四种启动模式

  • standard : 标准模式

    每次都创建活动的新实例。如果通过非Activity类型的Context启动活动，需要指定FLAG_ACTIVITY_NEW_TASK标记位，因为没有任务栈。

  • singleTop : 栈顶复用模式

如果活动在返回栈的栈顶，则不创建新实例，否则创建新实例。如果处于栈顶的活动启动本身，则会调用onPause、onNewIntent、onResume。

  • singleTask : 栈内复用模式

    如果活动在返回栈中，则不创建新实例，并将该活动之上的其他活动出栈，然后返回到栈顶。

  • singleInstance : 单例模式

    启用一个新的返回栈来管理活动。

    注：在 Androidanifest.xml 中注册活动时，在 标签中加入android:launchMode。

通过在Intent中设置标志位也可以为活动指定启动模式，并且优先级高于在AndroidManifest.xml文件中的设置。但无法为活动指定singleInstance模式。

2.2 Activity的Flags

  • FLAG_ACTIVITY_SINGLE_TOP : 为Activity指定 singleTop启动模式
  • FLAG_ACTIVITY_NEW_TASK : 为Activity创建一个新的任务栈
  • FLAG_ACTIVITY_CLEAR_TOP : 会清空任务栈
// 实现类似singleTask的启动方式，但活动会重新创建
Intent intent = new Intent(MainActivity.this, SecondActivity.class);
intent.addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP | Intent.FLAG_ACTIVITY_NEW_TASK);
